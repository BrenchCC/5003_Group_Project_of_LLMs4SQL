# 课程项目方向二：研究论文复现（SIGMOD/VLDB/ICDE 风格） 

## 项目概述
本方向旨在通过复现顶级数据库会议论文，深化你对数据库系统核心技术与研究方法的理解。你需要完成对一个完整研究工作的理解、实现与评估。

## 核心目标
选择一篇发表于顶级数据库会议（如 SIGMOD、VLDB、ICDE 或 EDBT）的研究论文进行复现。目标是理解论文核心思想与算法、完成（至少核心部分的）重新实现，并评估论文的技术贡献。该方向侧重批判性阅读、系统实现与实验验证能力。

**请注意**：所选论文需来自 SIGMOD、VLDB、ICDE 或同等水平的数据库领域会议（若会议级别不明确，需经导师批准）。

## 交付物与要求
项目需提交三部分内容，总评分为100%，具体构成如下：

### 1. 源代码（占总分 30%）
- **内容**：提交论文核心算法或系统组件的实现代码，附带测试脚本、合成/真实数据集及评估代码。
- **格式要求**：创建在线代码仓库（如 GitHub）。
- **仓库必须包含**：清晰的 `README.md` 文件，其中需详细说明：
    - 代码结构与模块划分
    - 项目依赖的软件、库与环境配置
    - 完整的代码运行与测试方法

### 2. 视频演示（占总分 20%）
- **内容**：录制 5-8 分钟的视频，讲解以下方面：
    1.  所选论文的核心思想
    2.  你的实现思路与关键决策
    3.  展示实验结果，如性能图表、正确性验证等
- **格式与提交要求**：
    - 使用屏幕录制工具（如 OBS、Zoom、Loom）制作。
    - 将最终视频上传至文件共享平台（如 Google Drive、Dropbox、OneDrive）。
    - **确保视频链接可公开访问**。
    - 视频链接需放入最终的书面报告中。

### 3. 书面报告（占总分 50%）
提交一份 PDF 格式的综合报告，文件命名必须为：`XXXX-report.pdf`。

**报告必须包含以下章节**：

- **小组信息**：列出所有成员的姓名及本科专业。
- **贡献说明**：明确划分每位成员的角色与具体贡献。
- **项目概述**：
    - 复现论文的标题、作者、发表会议
    - 选择该论文的动机
- **技术设计**：
    - 论文技术方法概述
    - 你自身的实现策略
    - 对原论文方法的偏差说明（如有）
- **算法/系统实现**：
    - 核心算法描述
    - 关键数据结构说明
    - 所采用的正确性验证方法
- **评估与结果**：
    - 实验设置（硬件、软件环境）
    - 所用数据集（合成或真实）描述
    - 评估的性能指标
    - （如适用）与原文结果的对比与分析
- **挑战与经验总结**：分享在复现过程中遇到的主要挑战及心得体会。
- **参考文献**：必须包含你所复现的论文的完整引用信息。
- **交付物链接**：提供以下有效链接：
    - 源代码仓库（如 GitHub）
    - 视频演示链接

## 重要原则
- **小组规模**：本项目由 **3-5人** 小组共同完成。
- **学术诚信**：所有代码与报告均需为原创。引用外部资源（包括代码片段、文献观点）时，**必须规范标注来源**。
- **截止日期**：请严格遵守课程规定的提交时间，逾期提交将按课程政策扣分。

## 项目价值
本项目旨在通过严谨的科学复现，锻炼你的批判性阅读、系统实现与实验验证能力——这些技能在现代数据管理研究与工程领域均具有重要价值。

请以好奇心为指引，力求研究深度、表达清晰与技术卓越。
